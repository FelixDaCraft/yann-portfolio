CupMetrics gère un concours de A à Z (inscriptions producteurs, paiements, espaces organisateur / jury / producteur, prix, expositions) avec une anonymisation déclenchée au paiement Stripe : chaque produit reçoit un code généré (initiales de catégorie + numéro unique en base) — les jurés ne voient que le code, jamais le producteur.

  • Intégration Stripe (checkout, webhooks) déclenchant l'anonymisation des produits
  • Espaces organisateur (résultats, règles de publication), jury (notation) et producteur (labels, résultats)
  • Moteur de résultats : scores, rangs par catégorie, labels par seuil, podium, Best in Show, statut « disqualifié » (anti-triche)
  • Génération PDF de synthèse, widget de palmarès embarquable, vitrine 3D (Three.js / R3F)
  • ~426 tests Vitest + Playwright ; CI/CD GitHub Actions (runner self-hosted) → build Docker → migration → mise en prod
  • Fork standalone Platinum CBD Cup et script de migration des données depuis le SaaS parent

PROJET
JLLM — github.com/FelixDaCraft/JLLM ↗ (bêta)
Type
Assistant RAG souverain sur le programme l'Avenir en commun
Statut
Bêta publique — opéré sur homelab
Contexte technique
Un LLM qui ne peut pas inventer : chaque réponse est générée uniquement depuis le texte du programme, sourcée et citée (chapitre · section), vérifiable. 99 % de fidélité mesurée au texte, refus systématique du hors-programme (7/7 aux tests d'injection).
Rôle au sein du projet
Conception, développement et exploitation — en autonomie complète
Tâches
  • Ingestion et découpage du programme en chunks sourcés (chapitre · section)
  • Retrieval hybride : embeddings BGE-M3 (dense + sparse + ColBERT) → Qdrant, fusion RRF
  • Reranking ColBERT puis génération encadrée par Mistral (souverain, température 0)
  • Garde-fous anti-hallucination et anti-injection, avec jeu de tests dédié
  • API FastAPI + interface de chat ; conteneurisation Docker ; exposition via Cloudflare Tunnel
Stack technique
Python · FastAPI · Qdrant · BGE-M3 · Mistral · Docker · Cloudflare Tunnel

PROJET
BabyMonitor — github.com/FelixDaCraft/v308-rtsp-hack ↗
Type
PWA self-hostée de surveillance bébé — reverse engineering + ML embarqué
Statut
En production — utilisé chaque nuit, opéré sur homelab
Contexte technique
Une caméra IP Xiaovv sans flux ouvert ni app fiable : j'ai démonté son protocole propriétaire — V308 sur TCP:8800, chiffrement AES-128-ECB — pour en faire une PWA de surveillance en temps réel, avec détection des pleurs par ML et notifications critiques sur iOS.
Rôle au sein du projet
Reverse engineering, développement et exploitation — en autonomie complète
Tâches
  • Reverse engineering du protocole (Ghidra / Frida) : handshake, chiffrement AES-128-ECB, framing vidéo
  • Décodage H.264 / HEVC et restreaming via go2rtc / ffmpeg
  • Détection ML embarquée des pleurs : YAMNet TFLite (classification audio AudioSet)
  • Notifications : Web Push PWA qui perce le mode silencieux iOS, fallback ntfy
  • Backend Node Express + better-sqlite3, conteneurisation Docker, reverse proxy Traefik
Stack technique
Python · YAMNet TFLite · Node Express · better-sqlite3 · go2rtc / ffmpeg · PWA Web Push · Docker · Traefik
